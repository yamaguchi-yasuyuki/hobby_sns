# タスク管理アプリ 詳細要件定義書（簡易版）

## 1. プロジェクト概要

### 1.1 プロジェクト名
**シンプルタスク管理アプリ**

### 1.2 背景・目的
- **背景**: 個人の日々のタスクを忘れずに管理したい
- **目的**: タスクの追加・一覧表示・完了チェックができる最小限のアプリ

### 1.3 スコープ
- WebアプリとしてMVPを実装
- 認証・DB連携は今回対象外（モックデータで動作）

---

## 2. 機能要件

### 2.1 機能一覧

| 機能ID | 機能名 | 要約 | MVP |
|--------|--------|------|-----|
| F-001 | タスク一覧表示 | タスクを一覧で表示 | Yes |
| F-002 | タスク追加 | 新規タスクを追加 | Yes |
| F-003 | タスク完了 | チェックボックスで完了/未完了を切り替え | Yes |
| F-004 | タスク削除 | タスクを削除 | Yes |

### 2.2 データ構造

**Task（タスク）**
- id: string（一意）
- title: string（タイトル）
- completed: boolean（完了フラグ）
- createdAt: string（作成日時 ISO8601）

---

## 3. UI/UX設計

### 3.1 画面構成
1. **ホーム（タスク一覧）**: タスク一覧 + 追加フォーム
2. シンプルな1ページ構成

### 3.2 レイアウト
- ヘッダー: アプリタイトル
- メイン: タスク追加フォーム + タスクリスト
- モバイルファースト、レスポンシブ対応

---

## 4. 技術スタック
- Next.js 16（App Router）
- TypeScript
- Tailwind CSS v4
- React 19
