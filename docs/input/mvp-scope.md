# MVP範囲

## MVP定義

### 目的
「投稿しなくていい」という価値提案が、実際に40代男性SNS未使用者の参入障壁を下げるかを検証する。フィードを眺めてコレクションを貯めるという体験が継続するかどうかを確認する。

### 対象ユーザー
身近な40代男性（友人・知人・同僚）100人程度。趣味がキャンプ・温泉・古着のいずれかに当てはまる人。

### 成功の定義
- MVPリリース後1ヶ月で、週1回以上アプリを起動するユーザーが30%以上
- 1人あたり平均コレクション保存数が5件以上

---

## スコープ

### MVP に含める ✅
| 機能 | 理由 |
|------|------|
| ホームフィード（ビジュアル・カード型） | コアバリューそのもの |
| ジャンル選択（初回オンボーディング） | フィードのパーソナライズに必須 |
| ジャンルフィルター | 見たいジャンルだけ見られることが重要 |
| カード詳細画面 | 外部コンテンツへの導線に必要 |
| 外部リンク遷移（アプリ内ブラウザ） | X・note・YouTubeへの導線 |
| 「〇人が気になっています」表示 | 社会性の最小単位として必要 |
| 「気になる」保存 | 継続使用のフックとして最重要 |
| コレクション一覧（カテゴリ別） | 保存の価値を実感してもらうため |
| メールアカウント登録 | コレクション保存に必要 |
| 匿名閲覧モード | 登録ハードルを下げるため |
| レスポンシブ対応（スマホ・PC・タブレット） | 全端末対応が必須要件 |

### MVP に含めない ❌
| 機能 | 理由 | 対応時期 |
|------|------|----------|
| フィードのAIパーソナライズ | MVPでは手動キュレーションで代替 | Phase 2 |
| コレクションへのメモ追加 | 最初はなくても成立する | Phase 2 |
| 新着通知 | まず使ってもらうことが先決 | Phase 2 |
| ダークモード | UIの完成度より機能検証を優先 | Phase 2 |
| PWA・オフライン対応 | 技術コストを抑えるため | Phase 2 |
| コレクションのエクスポート | 利用頻度を見てから判断 | 将来検討 |
| シーズン特集 | 運用体制が整ってから | 将来検討 |

## タイムライン（目安）[仮]

| フェーズ | 期間 | 内容 |
|----------|------|------|
| MVP | 〜2ヶ月 | コア機能のみで動くWebアプリ。コンテンツは手動キュレーション |
| Phase 2 | 3〜5ヶ月目 | フィードパーソナライズ・通知・PWA対応 |
| Phase 3 | 6ヶ月以降 | マネタイズ検証・コンテンツパートナーとの連携 |

---

## 制約事項
- コンテンツはMVP段階では手動キュレーション（自分または少数の協力者）
- デザインはシンプルさを優先。凝ったアニメーション・演出は後回し
- 認証はメールのみ（SNSログインはPhase 2以降）

## リスクと対策
| リスク | 対策 |
|--------|------|
| コンテンツ量が少なくてフィードが寂しい | 最初から質の高いコンテンツを50〜100件用意してからリリース |
| 「気になる」を押すだけで終わってしまう（外部へ飛ばない） | カード詳細の導線をわかりやすくデザインする |
| 登録せずに離脱するユーザーが多い | 匿名閲覧を充実させ、コレクション機能でアカウント登録を自然に促す |
